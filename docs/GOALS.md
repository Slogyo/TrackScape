# TrackScape Goals

TrackScape is a focused planning tool for model railway builders. It should make it easy to design layouts against real room, tabletop, track, and scenery measurements without forcing the user into a heavy CAD workflow.

The product will grow in small, testable phases. Each phase should leave the app useful and keep the underlying project simple enough to change.

## Phase 1 — Foundation

App shell, layout, theme system, unit system, tool state, default layers, canvas placeholder.

## Phase 2 — Basic Drawing

Rooms, tabletops, lines, rectangles, select/move, delete, snap to grid, object properties.

## Phase 3 — Measurement Accuracy

Real-world scale, metric/imperial conversion, track length, wire length, room/table dimensions, labels, scale presets for HO, N, OO, O.

## Phase 4 — Track Planning

Track library system such as Peco and Unitrack, flexible track, curves, turnouts, snapping, radius display, grade/elevation, track grouping.

## Phase 5 — Layout Objects

Buildings, platforms, roads, trees, scenery zones, water, tunnels, bridges, custom asset library.

## Phase 6 — Layers

Add/remove/rename layers, lock, hide/show, layer order, grouping, colour-coded layers.

## Phase 7 — Saving and Exporting

Save/load local projects, export JSON, image, PDF, print-friendly plans, thumbnails.

## Phase 8 — Advanced

3D preview, terrain editor, wiring planner, bill of materials, shopping list, sharing, cloud save.

## Product Principles

- Accuracy before visual complexity.
- Small modules and plain data structures.
- Fast, obvious controls with minimal interface decoration.
- Internal project data should remain portable and exportable.
- Add dependencies only when a clear feature needs them.
